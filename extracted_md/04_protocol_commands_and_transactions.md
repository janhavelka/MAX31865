# Protocol, Commands, and Transactions

## Interface Type and Pins

The MAX31865 uses an SPI-compatible interface. [Source: MAX31865 RTD-to-Digital Converter, p. 1]

The SPI-compatible interface uses four pins: SDO, SDI, CS, and SCLK. SDO is serial-data output, SDI is serial-data input, CS initiates and terminates a data transfer, and SCLK synchronizes data movement between the microcontroller and the MAX31865. [Source: MAX31865 RTD-to-Digital Converter, p. 16]

The serial clock is generated by the microcontroller and is active only when CS is low and during address and data transfer. [Source: MAX31865 RTD-to-Digital Converter, p. 16]

## SPI Modes and Clocking

The MAX31865 supports SPI modes 1 and 3. The datasheet note states that the CPHA bit polarity must be set to 1. [Source: MAX31865 RTD-to-Digital Converter, p. 16; MAX31865 RTD-to-Digital Converter, p. 17]

The inactive clock polarity is programmable in some microcontrollers, and the MAX31865 automatically accommodates either clock polarity by sampling SCLK when CS becomes active to determine the inactive clock polarity. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

Input data on SDI is latched on the internal strobe edge, and output data on SDO is shifted out on the shift edge. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

There is one clock for each bit transferred. Address and data bits are transferred in groups of eight, MSB first. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

## SPI Function Table

| Mode | CS | SCLK | SDI | SDO | Source |
|---|---|---|---|---|---|
| Disable Reset | High | Input disabled | Input disabled | High impedance | [Source: MAX31865 RTD-to-Digital Converter, p. 17] |
| Write | Low | CPOL = 1, SCLK rising; CPOL = 0, SCLK falling | Data bit latch | High impedance | [Source: MAX31865 RTD-to-Digital Converter, p. 17] |
| Read | Low | CPOL = 1, SCLK falling; CPOL = 0, SCLK rising | X | Next data bit shift | [Source: MAX31865 RTD-to-Digital Converter, p. 17] |

The function-table note defines CPOL as the clock polarity bit set in the microcontroller control register. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

The function-table note also states that SDO remains high impedance until 8 bits of data are ready to be shifted out during a read. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

## Address Byte and Read/Write Selection

Any transfer requires an address byte that specifies write or read, followed by one or more data bytes. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

The address byte is always the first byte transferred after CS is driven low. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

The MSB of the address byte, A7, determines whether following bytes are written or read. If A7 is 0, one or more byte reads follow the address byte. If A7 is 1, one or more byte writes follow the address byte. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

The register section states that registers are accessed using `0Xh` addresses for reads and `8Xh` addresses for writes. [Source: MAX31865 RTD-to-Digital Converter, p. 13]

Data is read from or written to the registers MSB first. [Source: MAX31865 RTD-to-Digital Converter, p. 13]

## Single-Byte Transactions

For a single-byte transfer, one byte is read or written and then CS is driven high. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

Figure 6, `SPI Single-Byte Read`, shows SDI carrying address bits A7 through A0 while SDO is high impedance, followed by SDO carrying data bits D7 through D0. [Source: MAX31865 RTD-to-Digital Converter, p. 18]

Figure 7, `SPI Single-Byte Write`, shows SDI carrying address bits A7 through A0 followed by data bits D7 through D0, with SDO high impedance. [Source: MAX31865 RTD-to-Digital Converter, p. 18]

## Multibyte Transactions

For a multiple-byte transfer, multiple bytes can be read or written after the address has been written. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

The address continues to increment through all memory locations as long as CS remains low. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

If data continues to be clocked in or out, the address loops from `7Fh/FFh` to `00h/80h`. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

Figure 8, `SPI Multibyte Transfer`, shows an address byte followed by data byte 0, data byte 1, and data byte N; it shows both write data on SDI and read data on SDO for the multibyte format. [Source: MAX31865 RTD-to-Digital Converter, p. 18]

In the multibyte transfer figure, the address byte precedes all data bytes while CS remains low; the write path is shown on SDI and the read path is shown on SDO. [Source: MAX31865 RTD-to-Digital Converter, p. 18]

## Invalid and Read-Only Access Behavior

Invalid memory addresses report an `FFh` value. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

Attempting to write to a read-only register results in no change to that register's contents. [Source: MAX31865 RTD-to-Digital Converter, p. 17]

## Transaction-Triggered Side Effects

When the conversion mode is `Normally Off`, writing 1 to the 1-shot bit starts a conversion. The conversion is triggered when CS goes high after writing 1 to the bit. [Source: MAX31865 RTD-to-Digital Converter, p. 13]

If a multibyte write is used for the 1-shot bit, the conversion is triggered when CS goes high at the end of the transaction. [Source: MAX31865 RTD-to-Digital Converter, p. 13]

If VBIAS is on as selected by the Configuration register, the RTD voltage is sampled when CS goes high and conversion begins. [Source: MAX31865 RTD-to-Digital Converter, p. 13]

The automatic fault-detection cycle time in the electrical table is measured from CS high to cycle complete. [Source: MAX31865 RTD-to-Digital Converter, p. 3]

## Documented Configuration Write Patterns

The PDFs do not define separate command opcodes beyond register write/read address bytes. Configuration behavior is controlled by writing bit patterns to the Configuration register. [Source: MAX31865 RTD-to-Digital Converter, p. 13; MAX31865 RTD-to-Digital Converter, p. 14]

| Configuration register write pattern | Documented action | Read meaning while bits are set | Source |
|---|---|---|---|
| `XXXX00XXb` in D3:D2 field | No action | Fault detection finished | [Source: MAX31865 RTD-to-Digital Converter, p. 14] |
| `100X010Xb` | Fault detection with automatic delay | Automatic fault detection still running | [Source: MAX31865 RTD-to-Digital Converter, p. 14] |
| `100X100Xb` | Run fault detection with manual delay, cycle 1 | Manual cycle 1 still running; waiting for user to write 11 | [Source: MAX31865 RTD-to-Digital Converter, p. 14] |
| `100X110Xb` | Finish fault detection with manual delay, cycle 2 | Manual cycle 2 still running | [Source: MAX31865 RTD-to-Digital Converter, p. 14] |

`X` is documented as don't care in the fault-detection cycle control table. [Source: MAX31865 RTD-to-Digital Converter, p. 14]

## MAXREFDES42 IO-Link Module Transactions

This section is not the bare MAX31865 SPI protocol. It is included because the MAXREFDES42 quick-start guide exposes MAX31865-related values through IO-Link indexes. [Source: MAXREFDES42# IO-Link RTD Temp Sensor Quick Start Guide (IQ2 IO-Link Master Version), p. 5]

| IO-Link index | Data meaning in quick-start guide | Read instructions or format | Source |
|---:|---|---|---|
| 261 | MAX31865 RTD code register value variable. The guide states that the RTD code register value is shifted by 1 bit, so the value read back needs to be divided by two. | Use index type `uint16`, enter index `261`, use data type `uint16`, and press Read in the On-request Data Read Request group. | [Source: MAXREFDES42# IO-Link RTD Temp Sensor Quick Start Guide (IQ2 IO-Link Master Version), p. 15; MAXREFDES42# IO-Link RTD Temp Sensor Quick Start Guide (IQ2 IO-Link Master Version), p. 16] |
| 268 | Ambient temperature in degrees C variable. | Enter index `268`, change Data type to hex, press Read, and convert the read hex value to float32. | [Source: MAXREFDES42# IO-Link RTD Temp Sensor Quick Start Guide (IQ2 IO-Link Master Version), p. 17; MAXREFDES42# IO-Link RTD Temp Sensor Quick Start Guide (IQ2 IO-Link Master Version), p. 18] |

The MAXREFDES42 guide gives an RTD-resistance example using index 261:

```text
RTD = (Code * 400) / 2^15 = (18174 / 2) * 400 / 2^15 = 110.925 Ohms
```

This example is specific to the MAXREFDES42 IO-Link readback context, where the guide says the readback value is shifted by 1 bit and divided by two. [Source: MAXREFDES42# IO-Link RTD Temp Sensor Quick Start Guide (IQ2 IO-Link Master Version), p. 16]

The MAXREFDES42 guide gives an ambient-temperature example: `Temp = 0x41E00E8A = 28.007099 = 28.01 degrees C`. [Source: MAXREFDES42# IO-Link RTD Temp Sensor Quick Start Guide (IQ2 IO-Link Master Version), p. 18]

## Protocol Gaps

The provided PDFs do not document CRC, checksum, parity, packet retries, command acknowledgments, device IDs, broadcast addressing, or interrupt-mask commands for the bare MAX31865 SPI interface; the documented SPI interface consists of address and data byte transfers to the listed registers. [Sources: MAX31865 RTD-to-Digital Converter, p. 13; MAX31865 RTD-to-Digital Converter, p. 16; MAX31865 RTD-to-Digital Converter, p. 17; MAX31865 RTD-to-Digital Converter, p. 18]
